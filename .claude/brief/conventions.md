# Conventions

- **Naming:** PascalCase for components, camelCase for functions and variables, kebab-case for files
- **Imports:** absolute imports from `@/` (configured in tsconfig)
- **Comments:** only when the *why* isn't obvious — avoid restating what the code does
- **No tests** for now — flag if a piece of logic is complex enough to warrant it

## Flag before proceeding

- Any third-party package additions
- Any deviation from the defined folder structure
